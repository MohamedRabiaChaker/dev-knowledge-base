Duplicating code instead of refactoring spreads bugs and makes maintenance harder, as changes require hunting through multiple copies with the risk of missing one.

Refactor repeated code into shared utilities, functions, or classes. Make it easy to reuse rather than copy. DRY—Don't Repeat Yourself—keeps codebases maintainable and reduces bugs.
