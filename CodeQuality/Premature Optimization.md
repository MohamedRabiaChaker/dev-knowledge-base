Optimizing code for speed or memory before you know where the bottleneck actually is wastes effort and sacrifices clarity, making code harder to read and maintain to solve a problem that doesn't exist yet.

Build for correctness and clarity first. Measure performance with profiling tools, establish KPIs to track whether your code meets actual requirements, identify real bottlenecks, then optimize those specific areas. Optimize based on data, not guesses.
